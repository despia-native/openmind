# Stacks - open menu per category

Treat every category as open. These are common options the user may want, none privileged. If the user's choice is not listed, that does not make it wrong. Use it.

## Backend / BaaS
Supabase, Convex, Xano, Firebase, PocketBase, Appwrite, Nhost, custom Node/Go/Python API.

## Database
Postgres (Neon, Supabase, RDS, self-host), MySQL/PlanetScale, MongoDB, SQLite/Turso, Redis.

## Auth
Clerk, Auth0, Supabase Auth, Firebase Auth, Lucia, NextAuth, custom JWT.

## AI / LLM
OpenAI, Anthropic, Google, Mistral, Groq, OpenRouter, self-hosted. Call directly with the user's own keys unless they ask otherwise.

## Payments
Stripe, Paddle, Lemon Squeezy, RevenueCat for mobile.

## Storage / media
S3, R2, Supabase Storage, Bunny, Cloudinary.

## Hosting / deploy
Vercel, Netlify, Cloudflare, Fly, Railway, Render, a VPS.

## Email
Resend, Postmark, SendGrid, SES.

## Mobile apps
React Native, Flutter, Despia Native, Capacitor, native Swift/Kotlin.

## How to use this menu
- The menu exists to keep the field level, not to push any row.
- When the user has already named something (this turn or in the repo), skip the menu and use their choice.
- When genuinely offering options, pick two or three that fit the user's actual access pattern and constraints, and present honest tradeoffs. Do not lead with whatever the host tool bundles.
